가설 카드 — 무엇을 믿고 가르는가

축 1 · 문제 진단: 같은 "막힘" 불만의 원인을 다르게 본다 — 준비 부재 / 안내 부재 / 예측 불가 · 축 2 · 개입 강도: B(보존) < A(중간) < C(재설계)

시안 A
준비 체크리스트형

가설 — 사용자는 준비물 없이 시작했다가 중간에 막힌다. 시작 전에 준비시키면 완주한다.

우선 타겟 — 상황 B(오랜만에 복귀, 구조가 낯선 사용자).

이기는 이유 — "미리미리 말해라" 리뷰에 정면 대응. 빗썸·코인원에 준비물 화면 선례가 있어 학습 비용 낮음.

리스크 — 거래 중에 급히 들어온 상황 A 사용자에겐 인트로가 한 단계 마찰.

검증할 것 — 체크리스트를 읽고 시작하면 STEP 4·5(신분증·계좌) 이탈이 실제로 주는가?

시안 B
현행 골격 + 안내 레이어형

가설 — 현행 4단계 구조는 문제없다. 각 막힘 지점에 안내가 없을 뿐이다. 골격을 보존하고 안내만 얹으면 충분하다.

우선 타겟 — 기존 사용자(멘탈모델 유지) + 구현·설득 비용 최소화.

이기는 이유 — "구조는 그대로, 안내만 더했다" — AS-IS/TO-BE 대비(R9)가 가장 깔끔하고 발표 스토리가 안전.

리스크 — 임팩트가 작아 보임. 막힘의 구조적 원인(예측 불가)은 풀지 못함.

검증할 것 — 시트형 안내(닫으면 사라짐)로도 막힘 해소가 충분한가?

시안 C
기한 타임라인형

가설 — 미루다 차단당하는 진짜 원인은 "언제까지 안 하면 무엇이 막히는지" 예측 불가다. 기한을 1급 정보로 올리면 미룸도 분노도 준다.

우선 타겟 — 상황 A(거래 중 도래, "지금 vs 나중" 선택이 급한 사용자).

이기는 이유 — 레퍼런스 공백 영역(선례 없음) — 유일하게 "우리가 새로 풀었다"고 말할 수 있는 시안. 토스의 돌봄 톤 + 빗썸 공식 5h/1h 정책을 화면화.

리스크 — 공포 소구 경계선(코인원 "반 협박" 반면교사). 카피와 색을 디자인으로 눌러야 함.

검증할 것 — 타임라인을 본 사용자가 "나중에 하기"를 골라도 기한 내 복귀하는가?

분기 매트릭스

행 = 분기 지점 · 열 = 시안. 확정 사항(진행 표시 A안·완료 화면·Voice)은 행에 없음 — 전 시안 공통.
